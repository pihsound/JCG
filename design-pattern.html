01
생성 패턴 (Creational)
20년 4회21년 3회 23년 2회24년 1회

객체 생성에 관한 패턴. 유연성을 제공하면서도 프로그램 구조에 큰 영향을 주지 않는다.

🏭 Abstract Factory
추상 팩토리
관련 객체 집합을 생성하는 인터페이스 제공. 구체 클래스 지정 없이 캡슐화.
이케아 스타일 공장 — '북유럽 세트' 선택 시 소파·테이블·조명 모두 북유럽 스타일로 나온다
★★★
🏗️ Builder
빌더
복잡한 객체의 생성 과정과 표현 분리. 동일 절차로 다른 결과.
서브웨이 샌드위치 — 빵→고기→채소→소스 순서는 같지만 내용물은 손님마다 다르다
🍕 Factory Method
팩토리 메서드
객체 생성을 서브클래스로 분리. Virtual-Constructor 패턴. 상위 인터페이스 정의, 하위가 결정.
피자 지점 — 본사는 만드는 방법만 정의, 뉴욕 지점·시카고 지점이 각자 피자 종류를 결정
★★★★ 최빈출
📋 Prototype
프로토타입
기존 객체를 복제(Clone)하여 새 객체 생성. 초기화 비용이 클 때 유용.
기획서 복사 — 팀장의 완성본을 복사 후 각자 수정. clone() 메서드 사용
👑 Singleton
싱글톤
인스턴스가 오직 하나만 생성되도록 보장. 어디서든 같은 인스턴스에 접근. private 생성자 + static 메서드.
대통령 — 대한민국에 대통령은 한 명. 전 국민이 '대통령'을 부르면 항상 동일한 한 사람
★★★★★ 최빈출
💡 암기법 — 생성 5종

앱빌팩프싱 = 앱스트랙트팩토리 / 빌더 / 팩토리메서드 / 프로토타입 / 싱글톤
"앱 빌더가 팩토리에서 프로젝트 싱글로 만들었다"

02
구조 패턴 (Structural)
22년 3회25년 1회25년 2회

클래스나 객체를 조합해 더 큰 구조를 만드는 패턴.

🔌 Adapter
어댑터 · Wrapper
호환 불가 인터페이스를 변환. 기존 코드 수정 없이 연결.
220V → 110V 어댑터 — 한국 충전기를 미국 콘센트에 꽂기 위한 변환기
★★★★ 빈출
🌉 Bridge
브리지
구현부에서 추상층을 분리하여 독립적으로 확장. 기능·구현을 별도 계층으로.
TV 리모컨 — 리모컨(추상)과 TV 기기(구현)를 독립적으로 확장 가능
★★★ 빈출
🌳 Composite
컴포지트
객체를 트리 구조로 구성. 개별·복합 객체를 동일하게 다룸.
파일·폴더 구조 — 폴더 안에 파일도, 폴더도 존재. 둘 다 '열기·삭제' 가능
☕ Decorator
데코레이터
객체에 동적으로 기능 추가. 래핑(Wrap)으로 확장.
스타벅스 토핑 — 아메리카노 + 샷추가 + 시럽 + 휘핑. 원래 커피 바꾸지 않고 감싸서 추가
🎛️ Facade
퍼사드
복잡한 서브시스템에 단순화된 인터페이스 제공.
홈시어터 리모컨 — '영화 시작' 버튼 하나로 프로젝터·앰프·블루레이·조명 모두 자동 제어
🪶 Flyweight
플라이웨이트
대량의 유사 객체를 공유하여 메모리 최소화.
게임 나무 수천 그루 — 나무 이미지 공유, 위치 정보만 따로 저장
🕵️ Proxy
프록시
다른 객체에 대한 대리 객체를 제공해 접근 제어. 보안 검증·지연 초기화·캐싱에 사용.
연예인 매니저 — 팬이 연예인(실제 객체)에 직접 접근하지 않고 매니저(프록시)를 통해 간접 접근
★★★★ 빈출
💡 암기법 — 구조 7종

어브컴데 퍼플프 = 어댑터 / 브리지 / 컴포지트 / 데코레이터 / 퍼사드 / 플라이웨이트 / 프록시

⚠️ Adapter vs Decorator vs Proxy 혼동 주의

패턴목적인터페이스핵심
🔌 Adapter인터페이스 변환변환 (호환 불가→가능)Wrapper. 두 클래스 연결
☕ Decorator기능 추가동일 유지기능을 감싸서 확장
🕵️ Proxy접근 제어동일 유지대리인. 접근을 중간에서 제어
03
행위 패턴 (Behavioral)
21년 2회22년 3회 23년 2회24년 3회

객체·클래스 간의 알고리즘 및 책임 분배에 관한 패턴. 클래스 간 상호작용하는 방법 정의.

⛓️ Chain of Responsibility
책임 연쇄
요청을 체인으로 연결. 처리될 때까지 다음으로 전달.
회사 결재 라인 — 팀장→부장→이사→대표. 누군가 처리할 때까지 올라간다
📡 Command
커맨드
요청을 객체로 캡슐화. Undo/Redo 구현에 핵심.
식당 주문서 — 손님(요청자)과 셰프(실행자) 분리. 주문서 취소·재주문 가능
📜 Interpreter
인터프리터
언어의 문법을 정의하고 해석. 문법 규칙을 클래스로 표현.
SQL 해석기 — SELECT * FROM users를 DB가 문법 규칙에 따라 해석·실행
🔁 Iterator
이터레이터
컬렉션 요소에 순차 접근하는 방법 제공. 내부 구조 노출 없이.
TV 채널 리모컨 — 채널 저장 방식 몰라도 '다음' 버튼으로 순차 탐색
🧑‍⚖️ Mediator
미디에이터
객체 간 복잡한 통신을 중재 객체로 단순화. 결합도 감소.
공항 관제탑 — 비행기들이 서로 직접 통신하지 않고 관제탑을 통해 교신
💾 Memento
메멘토
객체의 내부 상태를 저장하고 복원. Undo 기능 핵심.
Ctrl+Z — 문서 작업 실수 시 이전 상태로 복원. 캡슐화 위반 없이 상태 저장
🔔 Observer
옵저버
한 객체의 상태 변화 시 의존 객체들에게 자동 통지. 1:N 의존관계. Publish-Subscribe 패턴.
유튜브 구독 알림 — 크리에이터(Subject)가 영상 올리면 구독자(Observer) 전원에게 자동 알림
★★★★ 빈출
🚦 State
스테이트
객체의 내부 상태에 따라 행동 변경. 상태를 별도 클래스로 분리.
신호등 — 빨간불이면 '정지', 초록불이면 '출발'. 상태마다 완전히 다른 행동
♟️ Strategy
전략
알고리즘 군을 정의하고 캡슐화·교체 가능하게 함.
네비게이션 — 최단경로·최소요금·고속도로 중 알고리즘만 갈아끼우기
📝 Template Method
템플릿 메서드
알고리즘 골격을 상위클래스에서 정의. 세부 단계는 하위클래스에서 구현.
카페 음료 레시피 — '에스프레소→담기→추가재료' 틀은 같고 내용물만 달라짐
🚶 Visitor
비지터
객체 구조 변경 없이 새로운 연산 추가. 방문자 객체가 각 요소를 방문하며 연산 수행.
세금 감사관 — 건물 구조 바꾸지 않고, 감사관(방문자)만 바꿔서 세금·안전점검·임대료 평가 추가
★★ 빈출

"책 커버인이 미메옵이라고 전략적으로 템플릿 비전을 보여줬다"
