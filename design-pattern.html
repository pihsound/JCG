Factory Method
 생성 서브클래스가 인스턴스 결정, Virtual-Constructor 🍕 지점별 피자 ★★★★☆
Abstract Factory
 생성 관련 객체 집합 생성, 인터페이스 제공 🏭 이케아 스타일 공장 ★★★☆☆
Observer
 행위 1:N 의존관계, 자동 통지 🔔 유튜브 구독 알림 ★★★★☆
Adapter
 구조 Wrapper, 인터페이스 변환, 호환성 🔌 220V→110V 변환 ★★★★☆
Bridge
 구조 추상·구현 분리, 독립 확장 🌉 리모컨과 TV의 다리 ★★★☆☆
Proxy
 구조 대리 객체, 접근 제어, 캐싱 🕵️ 매니저(대리인) ★★★★☆
Decorator
 구조 동적 기능 추가, 래핑 ☕ 커피 토핑 추가 ★★★☆☆
Template Method
 행위 알고리즘 골격, Hook 메서드 📝 카페 음료 레시피 틀 ★★★☆☆
Visitor
 행위 구조 변경 없이 새 연산 추가 🚶 세금 감사관 방문 ★★☆☆☆

패턴목적기존 인터페이스핵심
🔌 Adapter
 인터페이스 변환 변환 (호환 불가 → 가능) Wrapper. 두 클래스를 연결
☕ Decorator
 기능 추가 동일하게 유지 기능을 감싸서 확장
🕵️ Proxy
 접근 제어 동일하게 유지 대리인. 접근을 중간에서 제어
Section 05
기출 Quiz
정답 보기를 클릭하세요 →
Q1
GoF 디자인 패턴의 3가지 분류를 쓰시오.
✅ 생성(Creational) 패턴, 구조(Structural) 패턴, 행위(Behavioral) 패턴
Q2
객체 생성 처리를 서브클래스로 분리해 캡슐화하는 패턴으로, Virtual-Constructor 패턴이라고도 하며 상위클래스에서 인터페이스를 정의하고 하위클래스에서 인스턴스를 생성하는 패턴은?
✅ Factory Method (팩토리 메서드)
Q3
한 객체의 상태가 바뀌면 그 객체에 의존하는 다른 객체들에게 자동으로 내용이 갱신되는 방식의 패턴은?
✅ Observer (옵저버)
Q4
어떤 객체에 직접 접근하기 어려운 상황에서 대신 요청을 전달하거나 접근을 제어하는 대리 객체를 두는 패턴은? 보안 검증, 지연 초기화, 원격 접근, 캐싱 등에 사용된다.
✅ Proxy (프록시)
Q5
구현부에서 추상층을 분리하여 각자 독립적으로 변형이 가능하고 확장할 수 있도록 하는 패턴으로, 기능과 구현을 두 개의 별도 클래스로 구현하는 패턴은?
✅ Bridge (브리지)
Q6
서로 호환되지 않는 두 클래스 사이에서 인터페이스를 변환해 함께 작동하도록 해주는 구조적 패턴으로, Wrapper 패턴이라고도 불리는 패턴은?
✅ Adapter (어댑터)
Q7
특정 클래스의 인스턴스가 하나만 생성되도록 보장하고, 어디서든 접근 가능하도록 하는 패턴은? (2023년 2회 기출)
✅ Singleton (싱글톤)
Q8
객체 구조 내의 요소에 대해 수행할 작업을 캡슐화하는 패턴으로, 객체 구조의 클래스를 수정하지 않고도 새로운 연산을 추가할 수 있게 해주는 패턴은? (2023년 2회 기출)
✅ Visitor (비지터)
Q9
Chain of Responsibility, Command, Iterator, Observer 패턴이 속하는 패턴 분류는? (2021년 2회 기출)
✅ 행위(Behavioral) 패턴
