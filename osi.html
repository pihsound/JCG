01
계층별 상세 설명
20년 1회21년 3회 22년 1회23년 1회
7
응용 계층 (Application Layer)
전송 단위: 데이터(Data)
사용자와 직접 상호작용

네트워크 서비스와 사용자 사이의 인터페이스를 제공하는 계층. 우리가 일상에서 사용하는 웹 브라우저, 이메일, 파일 전송이 모두 이 계층에서 동작한다.

HTTP — 웹 (80) HTTPS — 보안 웹 (443) FTP — 파일 전송 (20/21) SMTP — 이메일 발송 (25) DNS — 도메인 → IP (53) TELNET — 원격 접속 (23) SNMP — 네트워크 관리 (161) DHCP — IP 자동 할당 (67/68) POP3 — 이메일 수신 (110)
6
표현 계층 (Presentation Layer)
전송 단위: 데이터(Data)

데이터의 형식 변환·암호화·압축을 담당. 서로 다른 데이터 표현 방식을 사용하는 응용 프로세스에 독립성을 제공하고 암호화한다.

▸ 기출 포인트: "데이터 표현이 상이한 응용 프로세스의 독립성을 제공하고, 암호화한다" → 표현 계층

JPEG / GIF / PNG — 이미지 형식 MPEG — 동영상 압축 SSL/TLS — 암호화
5
세션 계층 (Session Layer)
전송 단위: 데이터(Data)

두 시스템 간의 통신 세션(연결)을 설정·유지·종료하는 계층. 대화 제어와 동기화를 담당한다.

SSH — 보안 원격 접속 TLS — 전송 보안 NetBIOS — 네트워크 서비스
4
전송 계층 (Transport Layer)
전송 단위: 세그먼트(Segment)
★★★ 빈출

종단간(End-to-End) 신뢰성 있는 데이터 전송을 보장. 흐름 제어·오류 제어·분할·재조합을 담당. 포트 번호를 사용하여 프로세스를 식별.

▸ 기출 포인트: "포인트 투 포인트의 신뢰성 있는 데이터 전송을 보장하며, 물리 계층의 오류를 감지하고 수정한다" → 데이터링크 계층 (헷갈림 주의!)

TCP — 신뢰성, 연결 지향 UDP — 비신뢰성, 빠름
3
네트워크 계층 (Network Layer)
전송 단위: 패킷(Packet)
★★★ 빈출

데이터를 목적지까지 가장 안전하고 빠르게 전달하는 기능. IP 주소를 기반으로 최적의 경로(라우팅)를 결정한다.

IP — 인터넷 프로토콜 ICMP — 오류/상태 보고 ARP — IP → MAC 변환 RARP — MAC → IP 변환 IGMP — 멀티캐스트 관리 OSPF — 링크 상태 라우팅 라우터
2
데이터링크 계층 (Data Link Layer)
전송 단위: 프레임(Frame)
★★★ 빈출

물리 계층에서 발생할 수 있는 오류를 감지하고 수정하여 신뢰성 있는 데이터 전송 보장. MAC 주소(물리 주소) 기반. CRC(Cyclic Redundancy Check)로 오류 감지.

Ethernet — LAN 표준 PPP — 점대점 연결 스위치 브리지
1
물리 계층 (Physical Layer)
전송 단위: 비트(Bit)

데이터를 전기적 신호(비트)로 변환하여 통신 매체(케이블)를 통해 전송. 0과 1로만 구성. 데이터의 내용이나 오류는 처리하지 않음.

통신 케이블 랜카드(NIC) 리피터 허브

03
TCP vs UDP
20년 1회24년 3회 25년 1회

둘 다 4계층(전송 계층) 프로토콜. TCP는 신뢰성, UDP는 속도가 핵심.

TCP
Transmission Control Protocol
연결 방식연결 지향
신뢰성보장 (재전송)
흐름 제어있음
순서 보장있음
속도느림
헤더 크기20~60 Byte
사용 예HTTP, FTP, SMTP
UDP
User Datagram Protocol
연결 방식비연결 지향
신뢰성미보장 (재전송 없음)
흐름 제어없음
순서 보장없음
속도빠름
헤더 크기8 Byte
사용 예DNS, SNMP, DHCP, 스트리밍
💡 암기법

TCP = 택배 — 받은 사람이 서명(ACK)해야 하고, 안 오면 다시 보냄
UDP = 전단지 — 그냥 뿌리고 끝. 받았는지 확인 안 함. 대신 빠름

자주 나오는 프로토콜 정리
21년 3회23년 1회 24년 3회
프로토콜계층포트설명
HTTP7 응용80웹 페이지 전송
HTTPS7 응용443SSL/TLS 기반 암호화 HTTP
FTP7 응용20(데이터), 21(제어)파일 전송
SMTP7 응용25이메일 발송
POP37 응용110이메일 수신
DNS7 응용53도메인명 → IP 주소 변환 (UDP 사용)
DHCP7 응용67/68IP 주소 자동 할당
SNMP7 응용161/162네트워크 장비 관리·모니터링
TELNET7 응용23원격 접속 (비암호화)
TCP4 전송—연결 지향, 신뢰성 보장
UDP4 전송—비연결 지향, 빠름
IP3 네트워크—비신뢰성·비연결성 패킷 전달
ICMP3 네트워크—네트워크 오류 보고·진단 (ping 명령)
ARP3 네트워크—IP 주소 → MAC 주소 변환
RARP3 네트워크—MAC 주소 → IP 주소 변환
OSPF3 네트워크—링크 상태 라우팅 프로토콜. 다익스트라 알고리즘
⚠️ ICMP 기출 포인트 (23년 1회)

ICMP: 네트워크 오류 메시지 전송·상태 진단용 프로토콜. 일반 데이터 전송에는 사용하지 않음. TCP, UDP와 함께 동작. ping 명령이 ICMP를 사용함.

⚠️ ARP vs RARP 혼동 주의

ARP = IP 주소를 알 때 → MAC 주소 알아내기
RARP = MAC 주소를 알 때 → IP 주소 알아내기
