비선점
FCFSFirst Come First Served — 선입 선출

먼저 도착한 프로세스를 먼저 처리. 구현이 단순하고 공정하지만, 실행시간이 긴 프로세스가 앞에 오면 전체가 기다려야 하는 호위 효과(Convoy Effect) 발생.

비선점
SJFShortest Job First — 최단 작업 우선
★★★

실행시간(버스트 타임)이 가장 짧은 프로세스에 CPU를 먼저 할당. 평균 대기시간이 이론적으로 최소. 단, 실행시간이 긴 프로세스는 계속 밀릴 수 있어 기아(Starvation) 문제 발생.

💡 스토리

은행 창구 빠른 업무 우선 — 복잡한 대출 상담 고객보다, 통장 잔액 조회만 하는 고객을 먼저 처리. 전체 평균 대기시간은 줄지만, 대출 상담 고객은 계속 기다려야 함.

비선점
HRNHighest Response-ratio Next — 최고 응답률 우선
★★★

SJF의 기아 문제를 보완. 대기시간이 길수록 우선순위가 높아짐. 우선순위 값이 클수록 먼저 실행.

📐 HRN 우선순위 공식
우선순위 = (대기시간 + 서비스시간) ÷ 서비스시간
서비스시간 = 실행시간(버스트 타임) / 대기시간이 길수록, 실행시간이 짧을수록 우선순위 높아짐

선점형 알고리즘
22년 3회24년 2회25년 2회
선점
RR (Round Robin)라운드 로빈 — 시간 할당량 순환
★★★★

일정한 시간 할당량(Time Quantum/Slice)만큼 CPU를 사용하고, 완료되지 않으면 준비 큐의 맨 뒤로 이동. 시분할 시스템에 가장 적합.

▸ Time Quantum이 크면 → FCFS에 가까워짐 / 작으면 → 문맥 교환 오버헤드 증가

💡 스토리

식당 번호표 시스템 — 한 테이블에 20분씩만 앉을 수 있고, 시간이 지나면 다시 줄 뒤로. 모든 손님이 공평하게 순서를 기다림.

선점
SRTShortest Remaining Time — 최단 잔류 시간
★★★

SJF의 선점 버전. 새로운 프로세스 도착 시, 현재 실행 중인 프로세스의 남은 시간과 비교해서 더 짧으면 선점.

▸ 이론적으로 평균 대기시간 최소. 기아 문제 있음. 문맥 교환 오버헤드 발생.

선점
다단계 큐 / 다단계 피드백 큐MLQ / MLFQ

다단계 큐(MLQ): 여러 개의 준비 큐를 두고, 각 큐마다 다른 알고리즘 적용. 프로세스가 큐 간 이동 불가.

다단계 피드백 큐(MLFQ): MLQ에서 큐 간 이동을 허용. 오래 기다린 프로세스를 상위 큐로 이동시키는 에이징(Aging) 기법 적용 → 기아 문제 해결.
