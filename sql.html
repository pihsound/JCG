01
DDL — 데이터 정의어
22년 3회20년 2회

DDL(Data Definition Language)은 데이터베이스 구조(스키마)를 정의하는 명령어. AUTO COMMIT이 적용되어 ROLLBACK 불가.

명령어설명주요 옵션
CREATE테이블·인덱스·뷰 등 생성PRIMARY KEY, FOREIGN KEY, NOT NULL, UNIQUE, DEFAULT, CHECK
ALTER테이블 구조 변경ADD COLUMN, DROP COLUMN, MODIFY, RENAME
DROP테이블·뷰 등 완전 삭제 (구조+데이터)CASCADE, RESTRICT
TRUNCATE테이블 데이터만 삭제 (구조 유지). ROLLBACK 불가—
 
-- CREATE TABLE: 기본 구조
CREATE TABLE 학생 (
    학번     VARCHAR(10)  PRIMARY KEY,
    이름     VARCHAR(20)  NOT NULL,
    학과코드  INT          REFERENCES 학과(학과코드),
    성적     DECIMAL(3,1) CHECK (성적 BETWEEN 0 AND 4.5),
    학년     INT          DEFAULT 1
);

-- ALTER: 컬럼 추가
ALTER TABLE 학생 ADD COLUMN 전화번호 VARCHAR(15);

-- DROP vs TRUNCATE
DROP TABLE 학생;      -- 테이블 완전 삭제 (구조+데이터)
TRUNCATE TABLE 학생;  -- 데이터만 삭제 (구조 남음, ROLLBACK 불가)
DROP vs DELETE vs TRUNCATE 비교
명령어분류구조 삭제ROLLBACK속도
DROPDDL✓ (구조+데이터 전부)불가빠름
TRUNCATEDDL✗ (데이터만)불가빠름
DELETEDML✗ (데이터만)가능느림
02
DML — 데이터 조작어
20년 2회22년 3회 24년 3회25년 1회

DML(Data Manipulation Language)은 데이터를 조회·삽입·수정·삭제하는 명령어. ROLLBACK 가능.

① SELECT — 데이터 조회

SELECT [DISTINCT] 컬럼명, ...
FROM   테이블명
[WHERE  조건]
[GROUP BY 컬럼명]
[HAVING 그룹조건]
[ORDER BY 컬럼명 ASC|DESC];
SELECT 실행 순서 (중요!)

FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY

💡 암기법

프왜그하셀오 — 프롬 왜어 그룹바이 하빙 셀렉트 오더바이

-- 집계함수 + GROUP BY 예시
SELECT 학과, COUNT(*) AS 학과별인원, AVG(성적) AS 평균성적
FROM   학생
WHERE  학년 >= 2
GROUP BY 학과
HAVING COUNT(*) >= 10  -- 그룹 조건 (WHERE는 개별 행 조건)
ORDER BY 평균성적 DESC;
집계함수설명NULL 처리
COUNT(*)전체 행 수NULL 포함
COUNT(컬럼)해당 컬럼 값이 있는 행 수NULL 제외
SUM(컬럼)합계NULL 제외
AVG(컬럼)평균NULL 제외
MAX / MIN최댓값·최솟값NULL 제외

② INSERT / UPDATE / DELETE

-- INSERT
INSERT INTO 학생 (학번, 이름, 학과코드) VALUES ('2024001', '홍길동', 10);

-- UPDATE
UPDATE 학생 SET 성적 = 4.0, 학년 = 3
WHERE 학번 = '2024001';

-- DELETE
DELETE FROM 학생
WHERE 성적 < 1.0;  -- ROLLBACK 가능 (DML이므로)

③ WHERE 절 조건 연산자

연산자설명예시
BETWEEN A AND BA 이상 B 이하 범위성적 BETWEEN 3.0 AND 4.5
IN (값1, 값2, ...)목록 중 하나와 일치학년 IN (3, 4)
LIKE '패턴'문자열 패턴 매칭. %=임의문자, _=1문자이름 LIKE '김%'
IS NULL / IS NOT NULLNULL 값 여부성적 IS NULL

03
DCL · TCL — 권한·트랜잭션 제어
20년 2회21년 3회

DCL: 사용자 권한 부여·회수 | TCL: 트랜잭션 시작·완료·되돌리기

-- DCL: GRANT (권한 부여)
GRANT SELECT, INSERT ON 학생 TO user1;
GRANT ALL PRIVILEGES ON 학생 TO user1 WITH GRANT OPTION; -- 재부여 가능

-- DCL: REVOKE (권한 회수)
REVOKE SELECT ON 학생 FROM user1;
REVOKE ALL PRIVILEGES ON 학생 FROM user1 CASCADE;

-- TCL
COMMIT;                      -- 트랜잭션 확정 (영구 저장)
ROLLBACK;                    -- 트랜잭션 취소 (이전 COMMIT 상태로)
SAVEPOINT sp1;               -- 중간 저장점 설정
ROLLBACK TO SAVEPOINT sp1;  -- 저장점으로만 되돌리기
명령어분류설명
GRANTDCL사용자에게 권한 부여. WITH GRANT OPTION = 재부여 허용
REVOKEDCL부여한 권한 회수. CASCADE = 연쇄 회수
COMMITTCL트랜잭션 확정. 이후 ROLLBACK 불가
ROLLBACKTCL최근 COMMIT 이후 변경 취소
SAVEPOINTTCL트랜잭션 내 중간 저장점 설정
04
JOIN — 테이블 결합
21년 2회24년 1회 25년 1회

두 개 이상의 테이블을 관련 컬럼을 기준으로 결합하여 데이터를 검색하는 연산.

INNER JOIN
내부 조인
두 테이블 모두에 존재하는 데이터만 반환. 가장 많이 사용. 조인 조건 불일치 행은 제외.
LEFT (OUTER) JOIN
왼쪽 외부 조인
왼쪽 테이블 전부 + 오른쪽 테이블 일치 행. 오른쪽에 없으면 NULL.
RIGHT (OUTER) JOIN
오른쪽 외부 조인
오른쪽 테이블 전부 + 왼쪽 테이블 일치 행. 왼쪽에 없으면 NULL.
FULL OUTER JOIN
완전 외부 조인
양쪽 테이블의 모든 행 반환. 불일치 행은 NULL로 채움.
CROSS JOIN
교차 조인
양쪽 테이블의 모든 행 조합. 결과 행 수 = A행 × B행. 카티션 곱.
SELF JOIN
자기 자신 조인
동일 테이블을 두 번 사용. 별칭(alias) 필수. 계층 구조 탐색에 사용.
-- INNER JOIN 예시 (기출 유형)
SELECT e.name, p.name AS project_name
FROM employee e INNER JOIN project p
ON e.p_id = p.p_id;

-- LEFT JOIN
SELECT s.학번, s.이름, e.과목명
FROM 학생 s LEFT JOIN 수강 e
ON s.학번 = e.학번;  -- 수강 안 한 학생도 출력, 과목명은 NULL

-- CROSS JOIN (카티션 곱) — 24년 3회 기출 유사
SELECT COUNT(*) FROM T1 CROSS JOIN T2
WHERE T1.NAME LIKE T2.RULE;
기출 문제 유형 — 24년 1회

① Non-Equi Join: '='이 아닌 비교 연산자(<, >, ≠ 등)를 사용하는 조인
② Equi Join: '=' 동등 연산자를 사용하는 조인. 외래키↔기본키 매칭
③ Natural Join: 동일한 이름의 컬럼을 자동으로 매칭. 중복 컬럼 하나만 표시

05
서브쿼리 · 인덱스
20년 2회24년 3회
-- 서브쿼리: SELECT 안에 SELECT
SELECT * FROM 학생
WHERE 학과코드 IN (
    SELECT 학과코드 FROM 학과
    WHERE 학과명 = '컴퓨터공학'
);

-- 인덱스 생성 (20년 2회 기출)
CREATE INDEX idx_name ON student (name ASC);
DROP INDEX idx_name;
인덱스(INDEX) 핵심

• 검색 속도 향상을 위한 자료구조 (책의 색인과 같은 역할)
• SELECT 속도 ↑, INSERT/UPDATE/DELETE 속도 ↓ (인덱스도 갱신해야 하므로)
• CREATE INDEX 인덱스명 ON 테이블명(컬럼명);
