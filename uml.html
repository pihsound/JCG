01
UML이란?
22년 3회 21년 3회

UML(Unified Modeling Language)은 소프트웨어 시스템을 시각화·명세·구축·문서화하는 표준 모델링 언어로, 다양한 다이어그램을 통해 복잡한 시스템을 이해하기 쉽게 표현한다.

UML의 4가지 특징

특징설명
가시화 언어다이어그램으로 시스템 구조·동작을 시각적으로 표현
명세화 언어표준화된 언어로 시스템 요구사항·기능 명세
구축 언어설계·구현에 도움, 객체지향 개발에서 핵심 역할
문서화 언어기능·구조·동작을 표준화된 형태로 문서화
02
구성요소 3가지
22년 3회 24년 3회

UML의 구성요소 = 사물(Things) + 관계(Relationships) + 다이어그램(Diagram)

💡 암기법

사관다 — 사물 + 관계 + 다이어그램. "사관학교 다녀온 UML"

사물(Things) 종류

종류설명예
구조사물시스템의 개념적·물리적 요소클래스, 유스케이스
행동사물시간·공간에 따라 변화하는 행위상호작용, 상태머신
그룹사물요소를 그룹으로 묶어 관리패키지
주해사물부가적인 설명·제약 조건주석, 노트
인터페이스오퍼레이션 세트를 정의하는 모델 요소«interface»
03
다이어그램 13종
20년 4회 21년 2회 25년 3회

① 구조 다이어그램 (6종) — 시스템의 정적 구조 표현

이름설명
클래스 ★★★객체 타입 정의 및 관계 도식화. 상속·연관·의존성 표현
객체시스템 내 객체와 관계를 특정 시점에 시각화
컴포넌트소프트웨어 논리적 구성요소와 의존성 표현
배치시스템의 물리적 배치와 구성요소 간 관계
복합체 구조클래스·컴포넌트의 내부 구조 및 상호작용
패키지 ★★패키지와 클래스 간 그룹화 및 관계 시각화
💡 암기법 — 구조 6종

클객컴 배복패 — 클래스·객체·컴포넌트 / 배치·복합체·패키지

② 행위 다이어그램 (7종) — 시스템의 동적 행위 표현

이름설명
유스케이스기능과 사용자 간 상호작용. 구성: Actor, Use Case, System
순차 ★★★객체 간 상호작용을 시간 순서로 표현. 생명선·메시지·활성객체
커뮤니케이션시간 흐름보다 객체 간 관계와 통신에 중점
상태객체 생명주기 동안의 상태 변화 표현
활동프로세스·워크플로우 표현. 비즈니스 프로세스 모델링
상호작용활동 다이어그램 + 순차 다이어그램의 혼합 형태
타이밍객체 행동과 시간에 따른 상호작용. 실시간 시스템
💡 암기법 — 행위 7종

유순커 상활상타 — 유스케이스·순차·커뮤니케이션 / 상태·활동·상호작용·타이밍

04
관계(Relationships) 6종
21년 3회 24년 3회 25년 1회
① 연관 관계
Association
─────→
두 클래스가 서로 관련. 가장 일반적. 실선 화살표.
② 의존 관계
Dependency
- - - →
한 클래스가 다른 클래스를 사용. 점선 화살표. 변경 시 영향.
③ 일반화 관계
Generalization
────▷
IS-A 관계. 상위 개념 포함. 빈 삼각형 화살표. 상속.
④ 실체화 관계
Realization
- - - ▷
인터페이스와 구현 클래스 간 관계. 점선 + 빈 삼각형. «interface» 구현.
⑤ 집약 관계
Aggregation
◇─────
약한 HAS-A. 전체 사라져도 부분은 독립 존재. 빈 마름모.
불고기 ◇ 간장
⑥ 합성 관계
Composition
◆─────
강한 HAS-A. 전체 사라지면 부분도 소멸. 채워진 마름모.
책상 ◆ 책상다리
집약 vs 합성 핵심 차이
구분집약 (◇ 빈 마름모)합성 (◆ 채워진 마름모)
강도약한 HAS-A강한 HAS-A
생명주기독립적 (전체 삭제 후에도 부분 존재)의존적 (전체 삭제 시 부분도 소멸)
05
클래스 다이어그램 문법
21년 3회22년 3회

클래스 박스는 클래스명 / 속성 / 오퍼레이션 3구역으로 구성. 각 멤버 앞에 접근제어자 기호를 붙인다.

기호접근제어자범위
+Public모든 클래스에서 접근 가능
-Private해당 클래스 내부에서만
#Protected해당 클래스 + 하위 클래스
~Package동일 패키지 내 클래스
💡 암기법

+ 공개  - 사유  # 보호  ~ 패키지
